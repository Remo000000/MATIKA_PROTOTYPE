# МАТИКА — система автоматического составления расписания (Django)

Веб-приложение для вуза с **изоляцией данных по организациям** (`Organization`): можно вести несколько независимых «тенантов» в одной базе (отдельные факультеты, аудитории, слоты, пользователи). Это не биллинг и не полноценный мультиарендный SaaS с поддоменами — а **модель тенанта в БД** + настройки по умолчанию (`DEFAULT_ORGANIZATION_SLUG`).

Подходит для демо и пилота; перед боевым запуском — отдельные шаги по безопасности, почте и инфраструктуре.

## Возможности

- Роли: администратор / преподаватель / студент (пользователь привязан к организации)
- Управление данными (факультеты, кафедры, группы, аудитории, дисциплины, слоты) в рамках организации администратора
- Генерация расписания (жадно + оптимизация) **по организации**; в лог и интерфейс попадают **примеры неразмещённых пар** (нет аудитории по вместимости / нет свободного слота)
- Просмотр расписания с фильтрами; экспорт Excel — только уроки своей организации
- Аналитика (Chart.js из локальных статик): загрузка преподавателей и аудиторий **полностью**, экспорт **CSV**, таблица последних запусков алгоритма
- Импорт CSV, экспорт расписания (Excel), сброс пароля по email (нужен SMTP в `.env`)
- Регистрация: студент **выбирает группу**, преподаватель — **кафедру**; пароль проходит те же проверки, что и в админке
- Статика **без обязательного CDN**: Bootstrap, Bootstrap Icons, Chart.js лежат в `static/vendor/` (удобно для закрытого контура)
- i18n: Русский / Қазақша / English

## Быстрый старт (локально, без Docker)

### 1) Установка зависимостей

```bash
python -m venv .venv
# Windows:
.venv\Scripts\activate
pip install -r requirements.txt
```

### 2) Переменные окружения

Скопируйте `.env.example` в `.env` и отредактируйте значения. Для **сброса пароля и реальных писем** задайте `EMAIL_BACKEND` и параметры SMTP (см. комментарии в `.env.example`).

### 3) Миграции и запуск

```bash
python manage.py migrate
python manage.py seed_demo
python manage.py runserver
```

Откройте `http://127.0.0.1:8000/`.

### Демо-доступы (после `python manage.py seed_demo`)

Почты в формате казахских имён, домен `@gmail.com` (демо).

> **Важно:** пароли ниже только для локальной демонстрации. Перед любым развёртыванием в сети смените пароли всех учётных записей и удалите или пересоздайте демо-данные.

- admin: `batima.tileikhan@gmail.com` / `admin12345`
- teacher / student: `имя.фамилия@gmail.com`, пароли `teacher12345` / `student12345`

После `seed_demo` конкретные почты берутся из общего пула имён (без пересечений).

**Если в базе ещё старые демо-почты** (`@matika.local`), выполните:

`python manage.py apply_kazakh_demo_identities`

## PostgreSQL

Проект настроен на PostgreSQL через `DATABASE_URL` (см. `.env.example`).

## Локализация

```bash
python manage.py makemessages -l ru -l kk -l en
python manage.py compilemessages
```

## Тесты и качество кода

```bash
pip install -r requirements.txt -r requirements-dev.txt
pytest
ruff check .
```

Перед продакшеном: `DEBUG=0`, сильный `SECRET_KEY`, настройка `CSRF_TRUSTED_ORIGINS` и почты, при необходимости в админке Django **Sites** укажите домен для корректных ссылок в письмах сброса пароля, выполните `python manage.py check --deploy`.
