# Формулы MATIKA: признаки слота, эвристика, штрафы планировщика

Здесь собраны **основные формулы** из кода (`scheduling/ml/predict.py`, `scheduling/services.py`) с пояснением, **зачем** каждая используется.

---

## 1. Вектор признаков слота (7 чисел)

Для каждого временного слота строится вектор **v = (v₀…v₆)** — вход и для эвристики, и для Keras-модели (если она обучена).

| Индекс | Обозначение в коде | Смысл | Как получается |
|--------|-------------------|--------|----------------|
| v₀ | `dow/7` | День недели (нормированный) | `day_of_week / 7` |
| v₁ | `period/12` | Номер пары (нормированный) | `min(period, 12) / 12` |
| v₂ | `fatigue` | Индекс усталости студентов | `SlotPedagogicalFeatures.student_fatigue_index`, по умолчанию 0.5 |
| v₃ | `survey` | Нагрузка по опросам | `survey_burden_index`, по умолчанию 0.5 |
| v₄ | `lms` | Активность в LMS (норм.) | `lms_activity_normalized`, по умолчанию 0.5 |
| v₅ | `history` | Историческая нагрузка слота | `historical_semester_load`, по умолчанию 0.5 |
| v₆ | `monday_morning` | Признак «понедельник, ранние пары» | — 1.0, если день недели = 1 и пара ≤ 2; иначе 0.0 |

Все v₂…v₅ ограничиваются отрезком [0, 1].

**Где используется:** прогноз «нежелательности» слота, обучение модели, страница «Анализ слотов», экспорт CSV.

## 2. Эвристика (без нейросети)

Если модель Keras недоступна или не загружается, используется **фиксированная формула** (детерминированная):

\[
Z_{\text{heur}} = \mathrm{clip}_{[0,1]}\bigl(
  0{,}38\cdot v_6 + 0{,}22\cdot v_2 + 0{,}18\cdot v_3 + 0{,}12\cdot(1 - v_4) + 0{,}10\cdot v_5
\bigr)
\]

**Смысл коэффициентов (в коде заданы явно):**

- **v₆ (понедельник утром)** — самый сильный вклад в «плохость» слота.
- **v₂ (усталость), v₃ (опрос)** — основные педагогические сигналы.
- **(1 − v₄)** — высокая активность в LMS **снижает** штраф (низкий LMS → выше вклад).
- **v₅ (история)** — прошлая загрузка слота.

**Важно:** в этой формуле **напрямую не участвуют v₀ и v₁** — день недели и номер пары влияют только через **v₆** (понедельник утром) и через то, какие **строки признаков** привязаны к слоту в БД.

**Где используется:** запасной прогноз при отсутствии TF/файла модели; базовая разметка при обучении, если нет `target_unfitness_label`.

## 3. Прогноз с Keras (если есть)

- Файл модели: `MEDIA_ROOT/scheduling_ml/slot_unfitness.keras`
- Для **MLP** подаётся один вектор **v**; на выходе одно число в [0, 1], затем обрезка.
- Для **Transformer** по неделе нужна матрица всех слотов; в отдельных ветках при отсутствии контекста снова используется эвристика.

## 4. Штрафы в планировщике (целые единицы)

### 4.1. «ML-штраф» слота

\[
\text{penalty\_units} = \mathrm{round}\bigl( \mathrm{ML\_PENALTY\_SCALE} \cdot Z \bigr), \quad \mathrm{ML\_PENALTY\_SCALE} = 12
\]

где **Z** — `predict_slot_unfitness` (уже из модели или эвристики).

**Где:** добавляется к стоимости выбора слота при жадной расстановке и в генетическом алгоритме (`ml_penalty_units`).

### 4.2. Предпочтения преподавателя к слоту

- Не в предпочтительный день недели: **+3**
- Не в предпочтительную пару: **+2**

(`_slot_penalty_teacher`)

### 4.3. Окна у студентов (пропуски между парами в один день)

- За каждую «дыру» между соседними занятыми парами величину **(gap − 1)** начисляется **+2 × (gap − 1)** (если в дне ≥ 3 пар).

### 4.4. Перегруз подряд идущими парами (≥ 3 подряд)

- Учитывается **overload_penalty**; при длинной серии «без окон» добавляются **+2 × (длина − 2)** за серии.

### 4.5. Предпочтения преподавателя к слоту в фитнесе ГА

- Не в предпочтительный день: **+3** за урок
- Не в предпочтительную пару: **+2** за урок
- Ранняя первая пара: **+1**, поздняя (≥ 6): **+2** за урок
- Плюс сумма **ml_penalty_units** по всем урокам

### 4.6. Итоговый fitness (мягкие штрафы, фрагмент)

\[
\text{fitness} = -1000\cdot \text{hard} - 10\cdot\text{windows} - 5\cdot\text{overload} - 3\cdot\text{pref} - 2\cdot\text{earlyLate} - 2\cdot\text{ml\_penalty}
\]

(жёсткие нарушения `hard` доминируют по масштабу).

## 5. Персонализация для студента (смещение вектора)

При наличии `schedule_preferences` (чувствительность к усталости/опросу, предпочтение утра) компоненты вектора **смещаются** (`apply_student_schedule_biases`) — не отдельная формула в одну строку, а пошаговые правки v₂, v₃, v₆.

---

## 6. 3D-график на странице анализа

- **Ось X** — усталость (v₂), **Y** — опрос (v₃), **Z** — прогноз нежелательности.
- **Поверхность** строится только из **эвристики** на сетке по (v₂, v₃), остальные компоненты зафиксированы (первый слот организации или нейтральный базис) — это **аналитическая** поверхность формулы из §2.
- **Точки** — реальные слоты; **Z** = полный прогноз (`predict_slot_unfitness`); в подсказке выводятся **все 7** компонентов вектора.
